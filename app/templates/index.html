1. 先区分三种使用方式

              

                
离线 Demo在本机运行，只绑定 127.0.0.1，默认读取 sample_data/。适合快速试用、课堂展示、无公网环境下的手动标注流程演示。

                
在线演示访问当前公开地址查看平台界面和流程。它只能读取演示服务器上的数据，不能直接访问访问者电脑或其他服务器的数据盘。

                
服务器部署把平台部署到团队自己的服务器或挂载数据盘后，服务器目录导入才能浏览该机器上的大规模图像、TSV/XLSX 和模型输出。

              

            

            

              
2. 快速启动

              
离线 Demo

              
cd /path/to/UAIV-Labeler
pip install -r requirements.txt
bash offline_demo/run_offline.sh

              
离线 Demo 默认打开 http://127.0.0.1:7860，只在当前电脑可见。

              
服务器部署

              
在团队服务器上准备 Python 环境后，进入平台目录并启动：

              
cd /path/to/UAIV-Labeler
pip install -r requirements.txt
bash scripts/start_public.sh

              
启动后访问：

              
http://<外部服务器IP>:7860

            

            

              
3. 数据放置

              
推荐把图像、TSV、XLSX 放在同一台服务器的数据盘，例如：

              
/data/UAIV/images
/data/UAIV/QA/questions.tsv
/data/UAIV/QA/results.xlsx

              
如果使用 QA 工具，建议将 QA 文件和配套图像放在同一数据目录下；平台会根据题目中的图像信息自动匹配，必要时也可以手动指定 QA 图像目录。

            

            

              
4. 公网访问

              
当前公开演示地址：

              
http://121.48.163.156:7860

              
其他服务器部署时，需要开放 7860 端口。云服务器还要在安全组/防火墙中放行 TCP 7860。

              
http://<server-ip>:7860

              
127.0.0.1 只代表服务器本机内部访问，不能作为发给外部用户的公开链接。

            

            

              
5. 域名与正式服务

              
长期公开建议使用 Gunicorn + Nginx。仓库已提供模板：

              
deploy/nginx_annotation.conf
deploy/uaiv-labeler.service
deploy/DOMAIN_DEPLOYMENT.md

              
有 root 权限后，可按 deploy/DOMAIN_DEPLOYMENT.md 配置域名、Nginx 反向代理和 HTTPS。

            

            

              
6. 注意事项

              
    
                
  • 公开演示可以用当前 Flask 服务；正式多人使用建议加登录认证。
    • 
                
  • 本地上传适合小规模 TSV/XLSX 或少量图片，大规模数据应使用服务器目录或挂载盘。
    • 
                
  • 如果要使用本地 YOLO、SegEarth、SAM 等模型，外部服务器也需要配置对应环境和权重。

4. 数据导入

              
服务器目录

              
适合直接使用平台所在服务器或挂载盘中的大规模影像数据，导入时仅建立索引，不重复复制原图。

              
    
                
  1. 选择“服务器目录”。
    2. 
                
  2. 输入路径或点击“浏览”。
    3. 
                
  3. 填写批次、飞行高度、经纬度、天气、场景和默认任务。
    4. 
                
  4. 点击“开始导入”或“使用当前目录”。
    5. 
              

              
电脑本地

              
适合从当前电脑选择少量影像进行试用或补充导入，平台会自动完成上传与索引。

              
导入完成后会在右上角弹出成功提示，并在工作台状态栏显示扫描数量、导入数量和跳过数量。

            


            

              
5. 图像列表与筛选

              
左下角图像列表显示图像 ID、场景、批次、天气、高度、任务、复核状态和数据来源。数据来源分为“实际数据”和“示例图”。筛选条件包括任务、场景、天气、批次、数据来源和飞行高度范围。筛选只作用于已经导入/已索引图像，用来缩小当前标注子集，不会重新读取目录。

            


            

              
6. Metadata

              
右侧 Metadata 面板显示当前图像的文件路径、批次、拍摄时间、飞行高度、经纬度、天气、场景、任务、宽高、来源目录等信息。若显示“未提取/未填写”，说明图像没有可读 EXIF/GPS，或导入时未填写默认值。

            


            

              
7. 人工标注

              
人工画框

              
在工具栏输入目标类别，例如 ship、vehicle、person，点击“人工画框”，然后在图像上拖拽生成 bbox。右侧目标框列表可修改类别与坐标。

              
添加目标框

              
该按钮会在当前图像中心新增一个默认框，宽约原图 18%、高约原图 14%，类别取工具栏输入框。它适合快速放置一个可编辑框；如果要按真实位置画框，请使用“人工画框”。

              
复核字段

              
右侧“标注复核”可编辑场景、环境、事件、QA、图像复原配对和城市结构理解。

            


            

              
8. 模型预标注

              
模型中心位于图像工作区下方。当前目标检测模型为 s2det-yolov8s，用于低空遥感目标检测预标注；SegEarth-OV 用于语义分割预标注。模型运行完成后会提示生成结果数量，即使没有检测到有效目标，也会给出完成提示。

              
自定义模型接入

              
如果团队已有检测、分割、OCR 或多任务模型，可以在模型中心填写模型名称、任务类型和服务地址，用于生成当前图像的初始标注，再由人工复核确认。

              
高级接入时，模型服务需接收当前图像信息，并返回标准标注结果，例如目标框、分割区域、OCR 文本、事件或 QA 条目。

              
{
  "objects": [
    {"label": "vehicle", "bbox": [120, 80, 60, 40], "score": 0.91, "status": "model:custom"}
  ],
  "segments": [],
  "ocr": [],
  "events": []
}

              
SAM3 为什么暂未直接接入？

              
SAM3 更适合带点、框或文本提示的交互式分割。后续会在提示交互和 GPU 服务模式稳定后，作为交互式分割工具接入平台。

10. 导出

              
顶部导出分为“服务器导出”和“本地下载”。服务器导出写入 exports/；本地下载由浏览器下载到当前电脑，VOC 会自动打包为 voc.zip。

              

                
                  
JSON完整 Metadata 与标注，适合归档和平台复用。

                  
COCO适合目标检测训练和评测。

                  
VOC每张图一个 XML，适合传统检测流程。

                  
QAJSONL 格式，适合事件理解和大模型 Benchmark。

                
              

            


            

              
11. 常见问题

              
为什么模型效果仍可能不好？

              
当前目标检测已换为 s2det-yolov8s，比原通用模型更贴近遥感检测流程；但不同数据集的视角、高度、天气和类别分布仍可能不同。后续可用平台复核数据继续训练 UAIV 专用模型。

              
为什么有些图没有经纬度？

              
平台会尝试读取 EXIF/GPS。若图像没有这些信息，且导入时没有填写默认经纬度/高度，就会显示为空。

              
为什么旧图一打开有框？

              
早期版本曾自动生成示例预标注。已写入的旧框不会自动删除，避免误删复核数据；可用“清除所有”后保存。

              
框坐标是原图坐标吗？

              
是。前端可能缩放显示大图，但保存和导出的 bbox 均为原图坐标。
