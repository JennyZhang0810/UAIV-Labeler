Low-altitude data production workbench

不是通用标注工具的另一个壳，而是面向低空数据生产的工作台。

UAIV-Labeler 连接多任务标注、Metadata 管理、模型预标注、人工复核、风险队列和 benchmark-ready 导出。它服务的不是单一 box 标注，而是一整条低空数据集生产流程。

为什么它不是一个普通标注页

CVAT 和 Label Studio 解决的是通用标注问题。UAIV-Labeler 解决的是低空遥感数据生产问题：图像来自服务器目录、任务是多任务混合、Metadata 是生产入口、标注之后还要走 QA、发布和 benchmark export。

Metadata-first

数据管理先于画框

飞行批次、天气、场景、来源路径、飞行高度和质量状态不是附属信息，而是工作流中心。

Multi-task

不只检测

同一工作台内支持 object grounding、OCR、事件 QA、场景与环境状态等多类标注任务。

Production

面向真实生产

支持批次组织、角色模式、风险队列、草稿状态、批量导出和多轮复核，而不是单次 demo。

Release-ready

导出不是终点

平台导出的结构要能进入后续 QA、Foundry、dataset packaging 和 benchmark 流程。

当前最值得看的不是功能清单，而是已经成型的四个工作面。

这些都直接来自你仓库里现成截图，而不是额外构造的宣传图。

标注工作面

围绕图像列表、任务工具、右侧属性面板和保存逻辑展开，适合真实多人生产，而不是单一图片 demo。

QA 与复核工作面

风险队列、状态区分和复核流程明确可见，强调“生产后复核”而不是一次性标完即结束。

Metadata 管理工作面

支持 altitude、scene、weather、source path 等信息进入数据生产主流程，适配低空数据的真实组织方式。

模型辅助工作面

平台不是只做人工操作，也包含 model pre-annotation、SAM-style segmentation、OCR 与自定义 backend 接口。

一个页面看懂当前平台最成熟的界面能力

如果观众只停留一分钟，下面这张表就足够解释平台已经具备哪些和低空数据生产直接相关的能力。

界面能力 当前状态 说明
Role-aware workspace ready 按 annotator / reviewer / admin / model operator 区分工作界面，减少多人生产时的干扰项。
Multi-task annotation ready 支持 detection、rotated box、polygon、OCR、event QA 和 image-level metadata review。
Risk queue and review flow ready 可按风险优先级组织复核工作，而不只是遍历式检查。
Model-assisted pre-annotation ready 支持 YOLO、SegEarth、SAM-style backends 和自定义 HTTP 模型接口。
Benchmark-ready export ready 导出格式面向后续训练、QA 和 benchmark packaging，而不是仅供页面展示。
